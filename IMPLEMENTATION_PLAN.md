## 子代理架构设计

### 架构概览

```
用户
  │
  ▼
┌─────────────────────────────────────────────┐
│              Main Controller                │
│         (main.cpp + WindowManager)          │
└─────────────────────────────────────────────┘
         │              │              │
    ┌────▼────┐   ┌────▼────┐   ┌────▼────┐
    │  Math   │   │   GUI   │   │  App    │
    │  Agent  │   │  Agent  │   │  Agent  │
    └─────────┘   └─────────┘   └─────────┘
```

### 子代理职责

#### 1. Math Agent (数学代理)

**目标**：负责所有几何计算和数学算法

**职责**：
- 曲线/曲面数学实现
- 求值算法
- 拟合/插值算法
- 几何变换

#### 2. GUI Agent (界面代理)

**目标**：负责所有UI组件和用户交互

**职责**：
- ImGui窗口实现
- 交互逻辑
- 视图控制
- 用户输入处理

#### 3. App Agent (应用代理)

**目标**：负责应用级功能和集成

**职责**：
- 文件操作
- 撤销/重做
- 命令系统
- 应用状态管理
